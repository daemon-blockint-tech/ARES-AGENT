## 9. Security Recommendations

### Immediate (High Priority)

1. **Add input validation to `CveBridge.enrich_findings`** ‚Äî This function has 22 downstream nodes, 0 upstream callers, and CC=9. It spawns a subprocess with user-supplied keywords. Add:
   - Keyword length limits
   - Character allowlist (alphanumeric + spaces)
   - Subprocess timeout enforcement

2. **Re-validate webhook URLs at dispatch time** ‚Äî `validate_webhook_url` checks at registration, but `dispatch_webhooks` sends without re-validation. DNS rebinding attacks can change the IP between registration and dispatch.

3. **Add sandboxing to bytecode analysis** ‚Äî `check_arbitrary_cpi`, `check_missing_owner_check`, and `CpiTracerDetector.analyze_edges` process untrusted bytecode. Malicious bytecode could exploit parser bugs.

### Medium Priority

4. **Audit `CVEEnricher._parse_cvedb_entry` (CC=17)** ‚Äî Highest complexity function parsing untrusted CVE data. Consider splitting into smaller functions with unit tests per branch.

5. **Add bounds checking to `MerkleTree.proof` (CC=8)** ‚Äî Generates Merkle proofs from untrusted finding data. Edge cases with empty or single-element trees need explicit tests.

6. **Validate Anchor discriminator computation** ‚Äî `build_anchor_instruction` computes `sha256("global:anchor_finding")` with 16 downstream nodes. Ensure the discriminator matches the on-chain program exactly.

### Low Priority

7. **Reduce proxy node count** ‚Äî 710 of 1,057 nodes (67%) are proxy/unresolved nodes. Many are from Rust macro expansions and trait method calls. Consider adding trait impl resolution to improve graph accuracy.

8. **Add cross-module call tracking** ‚Äî Only 1 cross-module call edge was detected (Python `SemanticMutator ‚Üí MutationResult`). Rust crate-to-crate calls are not tracked as cross-module because they share the same top-level module path.
